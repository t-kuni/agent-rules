# domain/services のコメントの書き方

このレイヤーのパブリック関数の関数コメントは以下の書式で記述する
理由： このレイヤーには複雑なロジックまたは再利用可能なロジックが含まれるため

```
// # 責務・責任範囲
// - この関数の責務。責任範囲
//
// # 想定ユースケース
// - どういう時に使われるのを想定しているか
//
// # 引数
// - 引数にどういう値が入力されることを想定しているか
//
// # 返り値
// - 返り値の説明
//     - errorの説明は省略する
```
